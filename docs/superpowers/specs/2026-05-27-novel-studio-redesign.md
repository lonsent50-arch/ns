# Novel Studio 全面重构设计规格

## 概述

将 Novel Studio 从单体工作台重构为路由式 SPA，新增首页书架、发布模块、导入功能、卷纲管理、记忆系统、状态追踪、AI 配置等完整功能体系。

## 架构决策

- **路由模式**：客户端 Hash 路由（`#bookshelf` / `#workspace/<id>` / `#publish/<id>` / `#ai-config`）
- **布局模式**：首页书架 → 点项目进入三栏工作台 → 发布模块独立页面
- **前后端分离度**：保持 Flask 单体部署，前端 JS 模块化拆分，API 层统一
- **存储**：localStorage + 后端 SQLite（每项目独立 DB）+ Supabase（云端同步）

## 路由表

| 路由 | 页面 | 说明 |
|------|------|------|
| `#bookshelf` | 首页书架 | 默认页，项目卡片网格 |
| `#workspace/<id>` | 三栏工作台 | 左栏导航树+中栏编辑器+右栏AI面板 |
| `#publish/<id>` | 发布上线 | 封面生成+简介+导出 |
| `#ai-config` | AI 配置 | 模型参数+System Prompt+用量统计 |

## 模块详细设计

### 1. 首页书架 (`#bookshelf`)

**布局**：
- 顶栏 56px：Logo + 搜索框 + 导入按钮 + AI配置按钮 + 设置按钮
- 分类标签行：全部 / 创作中 / 已完结 / 草稿（带计数）
- 项目卡片网格：CSS Grid `repeat(auto-fill, minmax(220px, 1fr))`
- 每张卡片：封面占位图（Canvas 渲染渐变色+书名首字）+ 书名 + 进度条 + 状态标签 + 最近编辑时间
- 最后两张卡片固定为"新建项目"和"导入作品"

**交互**：
- 点击项目卡片 → `#workspace/<id>`
- 右键/长按卡片 → 弹出菜单（重命名/复制/删除/发布）
- 新建卡片 → 弹窗（书名+简介+分类+目标字数）
- 导入卡片 → 导入向导

### 2. 三栏工作台 (`#workspace/<id>`)

**布局**：CSS Grid `280px 1fr 320px`

**左栏（280px）**：
- 顶部 `< 返回书架` 面包屑
- 书名显示
- 卷→章树形结构：卷节点可折叠，章节点高亮当前编辑
- 每章右侧状态点：🟡草稿 / 🔵润色中 / 🟢已完成（点击循环切换）
- 底部操作：`+ 新建卷` `+ 新建章`
- 底部模块入口：角色管理、世界观设定、写作统计、记忆系统

**中栏（flex, 编辑器）**：
- 章节标题输入框 + 保存状态指示
- 工具栏（图标按钮+hover tooltip）：AI续写、全屏、润色、健康度、删除
- 编辑器正文区：contenteditable div, max-width 900px, 居中, 行高2, 字号17px
- 底部：字数统计 + 章节导航（◀ 1/35 ▶）
- 粘贴自动剥离富文本

**右栏（320px）**：
- AI 辅助面板，可折叠卡片
- 快捷操作按钮组：续写/润色/写本章/扩写
- 当前活跃角色面板
- 世界观约束提示
- 实时写作统计
- POV 锁定控件
- 可切换为记忆系统面板

### 3. 发布模块 (`#publish/<id>`)

**布局**：两栏（左侧预览 50% + 右侧配置 50%）

**封面生成**（纯前端 Canvas 实现）：
- 6 种风格模板：玄幻、都市、科幻、古风、轻小说、暗黑
- 每种模板：CSS 渐变背景 + SVG 装饰 + 排版规则
- Canvas 渲染为 3:4 比例 PNG，可下载
- 配置项：书名、作者、风格、配色方案
- `生成封面` 按钮随机组合，`重新生成` 切换

**简介生成**（AI）：
- 基于全书大纲+前几章内容
- 可选风格：热血/悬念/温情/史诗
- 可选字数：100字/200字/300字

**导出格式**：
- ePub（含封面+目录+全部章节+元数据）
- TXT 纯文本
- PDF 格式化
- HTML 单文件

### 4. 导入功能

**入口**：书架页顶栏 `📂 导入` 按钮 + 书架网格导入卡片

**三步向导**：
- 步骤1：拖拽/选择文件（.txt .md .docx），显示最近文件
- 步骤2：AI 智能解析文档结构，识别卷/章分界，预览结果可手动调整
- 步骤3：确认导入，命名项目，选择新建或追加到已有项目

**文件解析**：
- .txt / .md：前端读取文本，发送到后端 AI 解析
- .docx：后端 python-docx 提取文本后 AI 解析
- 分章策略：正则匹配 + AI 语义识别双重保障

### 5. 卷纲管理

集成在工作台左栏的卷→章树中。

**操作**：
- 新建卷：行内输入卷名，自动创建第1章占位
- 新建章：在当前卷末或选中卷后添加
- 章节拖拽排序，支持跨卷移动
- 右键菜单：重命名/移动/删除/修改状态

**后端 API 扩展**：
- 在现有 outline 表基础上增加 `volume` 字段
- 或新建 `volumes` 表，关联 project_id

### 6. 状态追踪

- 每章三种状态：草稿 → 润色中 → 已完成（点击循环切换）
- 全书进度条：已完成章数/总章数 + 已完成字数/目标字数
- 顶部面包屑区显示全局进度

### 7. 记忆系统

入口在工作台左栏底部，在右栏面板区域渲染（三个 Tab）：

- **角色关系图谱**：Canvas 力导向图，节点=角色，连线粗细=互动频率，颜色=关系性质，节点大小=出场次数
- **伏笔追踪**：时间线列表，每条伏笔含描述+埋设章节+揭示章节+状态
- **关键事件时间线**：纵向时间轴，AI 提取每章关键事件

数据来源：AI 自动扫描全文提取 + 用户手动编辑

### 8. AI 配置 (`#ai-config`)

独立页面，包含：
- 默认模型选择 + API Key + Base URL
- 模型参数滑块：Temperature / Top P / Max Tokens
- System Prompt 编辑器（Textarea）
- 场景专属配置：续写/润色/大纲/封面/简介各用不同参数
- 用量统计：今日/本月调用次数、Token 消耗、预估费用
- 配置保存到 localStorage + 后端用户表
- API Key 加密存储
